# Supabase Setup for Registration + Training Access + Learner Progress

This project now supports:
- account registration (`register.html`)
- admin approval (`learner_profiles.approved`)
- protected training access
- per-user course completion tracking and badge count on dashboard

## 1) Add Supabase keys

Edit `assets/js/auth-config.js` and set:
- `supabaseUrl`
- `supabaseAnonKey` (anon/public only, never service_role)

## 2) Create profile table + trigger

Run in SQL Editor:

```sql
create table if not exists public.learner_profiles (
  id uuid primary key references auth.users(id) on delete cascade,
  full_name text not null,
  parish_team text not null,
  role_type text not null check (role_type in ('responsible', 'to-be-responsible')),
  age_group text not null check (age_group in ('6-8', '9-12', '13-15', '16-18')),
  phone text not null,
  email text not null unique,
  age integer not null check (age between 6 and 70),
  notes text default '',
  approved boolean not null default false,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create or replace function public.set_updated_at()
returns trigger
language plpgsql
as $$
begin
  new.updated_at = now();
  return new;
end;
$$;

drop trigger if exists trg_learner_profiles_updated_at on public.learner_profiles;
create trigger trg_learner_profiles_updated_at
before update on public.learner_profiles
for each row execute function public.set_updated_at();

create or replace function public.handle_new_user()
returns trigger
language plpgsql
security definer
set search_path = public
as $$
begin
  insert into public.learner_profiles (
    id, full_name, parish_team, role_type, age_group, phone, email, age, notes, approved
  )
  values (
    new.id,
    coalesce(new.raw_user_meta_data ->> 'full_name', ''),
    coalesce(new.raw_user_meta_data ->> 'parish_team', ''),
    coalesce(new.raw_user_meta_data ->> 'role_type', ''),
    coalesce(new.raw_user_meta_data ->> 'age_group', ''),
    coalesce(new.raw_user_meta_data ->> 'phone', ''),
    lower(coalesce(new.email, '')),
    coalesce((new.raw_user_meta_data ->> 'age')::int, 0),
    coalesce(new.raw_user_meta_data ->> 'notes', ''),
    false
  )
  on conflict (id) do update set
    full_name = excluded.full_name,
    parish_team = excluded.parish_team,
    role_type = excluded.role_type,
    age_group = excluded.age_group,
    phone = excluded.phone,
    email = excluded.email,
    age = excluded.age,
    notes = excluded.notes;

  return new;
end;
$$;

drop trigger if exists on_auth_user_created on auth.users;
create trigger on_auth_user_created
after insert on auth.users
for each row execute function public.handle_new_user();
```

## 3) Create completion/badge progress table

Run in SQL Editor:

```sql
create table if not exists public.learner_course_completions (
  id bigint generated by default as identity primary key,
  user_id uuid not null references auth.users(id) on delete cascade,
  training_id text not null,
  training_title text not null,
  module_id text not null,
  module_title text not null,
  badge_code text not null,
  completed_at timestamptz not null default now(),
  unique (user_id, training_id)
);

create index if not exists idx_learner_course_completions_user
  on public.learner_course_completions(user_id);
```

## 4) Enable RLS + policies

Run in SQL Editor:

```sql
alter table public.learner_profiles enable row level security;
alter table public.learner_course_completions enable row level security;

drop policy if exists "read own profile" on public.learner_profiles;
create policy "read own profile"
on public.learner_profiles
for select
to authenticated
using (auth.uid() = id);

drop policy if exists "update own profile" on public.learner_profiles;
create policy "update own profile"
on public.learner_profiles
for update
to authenticated
using (auth.uid() = id)
with check (auth.uid() = id);

drop policy if exists "read own completions" on public.learner_course_completions;
create policy "read own completions"
on public.learner_course_completions
for select
to authenticated
using (auth.uid() = user_id);

drop policy if exists "insert own completions" on public.learner_course_completions;
create policy "insert own completions"
on public.learner_course_completions
for insert
to authenticated
with check (auth.uid() = user_id);

drop policy if exists "update own completions" on public.learner_course_completions;
create policy "update own completions"
on public.learner_course_completions
for update
to authenticated
using (auth.uid() = user_id)
with check (auth.uid() = user_id);
```

## 5) Auth settings

In Supabase Authentication settings:
- Enable Email/Password
- Set Site URL + Redirect URLs for your GitHub Pages domain

## 6) Approval workflow

In Table Editor > `learner_profiles`:
- new users appear with `approved = false`
- set `approved = true` to unlock training access

## 7) How progress works now

- User opens a training page and clicks **Mark Training as Completed**
- Row is saved in `learner_course_completions`
- Dashboard shows:
  - learner profile info
  - completed trainings list
  - badge count
  - badge wall
  - progress percentage

## 8) Admin tracking query (who completed how many)

```sql
select
  p.full_name,
  p.email,
  p.parish_team,
  p.approved,
  count(c.id) as completed_courses
from public.learner_profiles p
left join public.learner_course_completions c
  on c.user_id = p.id
group by p.id, p.full_name, p.email, p.parish_team, p.approved
order by completed_courses desc, p.full_name asc;
```
